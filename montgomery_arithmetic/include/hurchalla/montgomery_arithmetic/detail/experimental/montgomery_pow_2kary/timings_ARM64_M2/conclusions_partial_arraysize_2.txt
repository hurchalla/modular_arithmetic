128 half gcc
noasm really wants 1u - it does 5.5% worse with 1y than 1u
asm wants 1y - it does 2.5% worse with 1u.
    The reason for 1y working better for asm, is that in ImplRedc.h, when
    HURCHALLA_ALLOW_INLINE_ASM_REDC is defined (this issue probably is
    ARM64 gcc only), gcc will use the LowuopsTag ARM64 inline asm if we
    specify 1u, and it produces horrible perf with this inline asm when
    doing 128 half.  I suspect a gcc bug.
So what we should do is
#if defined(HURCHALLA_ALLOW_INLINE_ASM_REDC) \
    || defined(HURCHALLA_ALLOW_INLINE_ASM_ALL)
  use 1y
#else
  use 1u
#endif
